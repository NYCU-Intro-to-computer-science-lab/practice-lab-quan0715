# p4: McCarthy 91 函數 (F91)

## 題目描述

McCarthy 是一位著名的資訊科學家。他定義了一個遞迴函數叫做 **f91**。

這個函數輸入一個正整數 N，並依據以下規則回傳一個正整數：

1. **如果 N ≤ 100**，則 `f91(N) = f91(f91(N+11))`
2. **如果 N ≥ 101**，則 `f91(N) = N - 10`

請你實作這個遞迴函數。

## 輸入格式

一個正整數 N (1 ≤ N ≤ 1000000)

## 輸出格式

輸出一個整數，表示 f91(N) 的值

## 範例

### 輸入 1
```
50
```

### 輸出 1
```
91
```

### 說明
- `f91(50) = f91(f91(61))`
- `f91(61) = f91(f91(72))`
- `f91(72) = f91(f91(83))`
- `f91(83) = f91(f91(94))`
- `f91(94) = f91(f91(105))`
- `f91(105) = 95`
- `f91(94) = f91(95)`
- `f91(95) = f91(f91(106))`
- `f91(106) = 96`
- `f91(95) = f91(96)`
- ... 經過多次遞迴，最終得到 91

---

### 輸入 2
```
110
```

### 輸出 2
```
100
```

### 說明
因為 110 ≥ 101，所以 `f91(110) = 110 - 10 = 100`

---

### 輸入 3
```
100
```

### 輸出 3
```
91
```

## 有趣的性質

McCarthy 91 函數有一個有趣的特性：
- **對於所有 N ≤ 100**，`f91(N) = 91`
- **對於所有 N ≥ 101**，`f91(N) = N - 10`

## 限制與要求

- 時間限制：1 秒
- 記憶體限制：256 MB
- **必須使用遞迴**: 你的程式碼中必須包含名為 `f91` 的遞迴函數
- **禁止使用迴圈**: 不可使用 `for`、`while`、`do` 等迴圈關鍵字

## 提示

- 函數簽章建議：`int f91(int n)`
- 記得處理兩個 base cases:
  - `n >= 101` 時，直接回傳 `n - 10`
  - `n <= 100` 時，需要遞迴呼叫兩次
- 函數內部會形成深度遞迴，但最終都會收斂
